# 🧠 知识复利引擎 — 让 SOLO 拥有持久记忆能力

> **作者：艾迪**
> **GitHub：https://github.com/theming028/skills/tree/main/knowledge-compound**

---

## 💡 灵感来源

使用 SOLO 的过程中，我发现一个核心痛点：**每次会话都是一次新的开始**。

每次启动新会话，SOLO 都是"失忆"的——我之前告诉过它的偏好、决定过的方案、踩过的坑，它完全不记得。这意味着：
- 同样的配置信息要反复交代
- 同样的决策要重新解释
- 踩过的坑下次还会再踩

知识如果不能积累，就没有复利效应。**所以我想让 SOLO 拥有记忆能力。**

---

## ✨ 作品简介

知识复利引擎是一个 Skill，它让 SOLO 能够**自动记住关键信息**并在**合适的时机主动调取**。

它的工作方式很简单：

1. **自动捕获** — 当用户说出关键信息（偏好、决策、项目背景、经验教训）时，SOLO 自动识别并记录
2. **知识复利三问过滤** — 只有值得长期记住的信息才会被保存
3. **智能召回** — 新会话启动时主动加载相关记忆，匹配的话题自动唤起
4. **可视化记忆库** — 用户可以随时查看、搜索、导出自己的记忆库

## 🔧 功能特点

### 🎣 自动捕获（无需手动操作）

SOLO 会识别以下场景并自动记录：

| 触发词 | 记录类型 | 示例 |
|--------|---------|------|
| "我喜欢/我习惯" | 偏好 | "我喜欢使用 pnpm 而不是 npm" |
| "决定了/就选" | 决策 | "就选 PostgreSQL 作为数据库" |
| "这个项目" | 项目上下文 | "这个项目是电商平台，用 Next.js" |
| "关键是/需要注意的是" | 经验 | "关键是支付回调要做幂等处理" |
| "踩过坑/遇到过问题" | 教训 | "踩过坑：MySQL 8.0 默认时区问题" |

### 🧪 知识复利三问（质量过滤器）

每条记忆存储前，自动通过三问筛选：
1. ❓ 未来会被多次用到吗？
2. ❓ 和其他知识有关联吗？
3. ❓ 用户会感谢我记住吗？

**≥2 个"是" → 存储 | <2 个"是" → 抛弃**

### 🔍 智能召回（多维评分）

记忆召回时，综合以下维度评分排序：
- 关键词匹配度（40%）
- 上下文相关性（30%）
- 标签匹配（20%）
- 访问频率（10%）

### 📊 记忆管理（8 个 CLI 命令）

```bash
# 存储一条记忆
python3 memory_bank.py store --type preference --content "..." --tags "tag1,tag2"

# 智能召回（自动感知上下文）
python3 memory_bank.py recall

# 搜索记忆
python3 memory_bank.py search --keyword "数据库" --limit 5

# 查看统计
python3 memory_bank.py stats

# 导出记忆库
python3 memory_bank.py export

# 清理 90 天未使用的记忆
python3 memory_bank.py cleanup
```

## 💪 技术亮点

1. **零外部依赖** — 纯 Python 3 标准库实现，无需 pip install
2. **隐私优先** — 所有数据存储在本地 JSON 文件，不上传任何信息
3. **智能去重** — 词重叠相似度算法（阈值 0.85）避免重复存储
4. **自动清理** — 90 天未访问的记忆自动归档，库大小可控
5. **从简原则** — 非侵入式设计，不改变用户现有工作流

---

## 🧠 设计哲学

> **真正的智能，是记住不该忘记的东西。**

这个 Skill 的设计围绕三个核心理念：
1. **最小化摩擦** — 自动工作，无需用户主动操作
2. **最大化价值** — 严格过滤，只保留真正值得记住的信息
3. **保护隐私** — 用户数据用户自己掌控
