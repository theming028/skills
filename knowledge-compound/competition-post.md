## 💡 灵感来源

使用 SOLO 的过程中，我发现一个核心痛点：**每次会话都是一次新的开始**。

每次启动新会话，SOLO 都是"失忆"的——我之前告诉过它的偏好、决定过的方案、踩过的坑，它完全不记得。这意味着：

- 同样的配置信息要反复交代
- 同样的决策要重新解释
- 踩过的坑下次还会再踩
- 累积的思考深度每次都从零开始

知识如果不能积累，就没有复利效应。**所以我想让 SOLO 拥有真正的记忆能力。**

---

## ✨ 作品简介

知识复利引擎是一个 Skill，它让 SOLO 从"每次会话都是新的开始"升级为**"越用越懂你"**的智能助手。

它提供 **5 层渐进激活能力**，日常无感运行，复杂任务自动升级：

```
L0 ─ 开机仪式（仅首次）    → 认识你，建立基础档案
L1 ─ 跨会话记忆（每次）   → 自动加载上下文，不用重复解释
L2 ─ 深度思考（任务时）   → OADA 闭环保证决策质量
L3 ─ 自我验证（输出后）   → 元认知自检，确保交付可靠
L4 ─ 持续进化（始终）     → 从纠正中学习，越用越聪明
```

## 🔧 核心能力

### 🎬 L0：开机仪式 — 首次会话自动触发

安装后第一次对话，SOLO 会自动启动欢迎仪式：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  🧠 知识复利引擎 v3.0 已激活
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

初次见面！让我先认识你，只需 30 秒：

① 你主要使用什么技术栈或工具？
② 你目前在做的项目是什么？
③ 你希望我了解你的哪些偏好或习惯？

（告诉我后，我会记住。以后每次新会话你都不用再重复。）
```

回答后自动存储，**仅此一次**，从此告别重复自我介绍。

---

### 🔄 L1：跨会话记忆 — 每次会话自动加载

每次新会话启动时，SOLO 自动输出你的上下文摘要：

```
📋 [知识复利引擎] 已加载你的上下文：
├─ 技术栈：React · TypeScript · pnpm
├─ 项目：电商平台（Next.js + PostgreSQL）
├─ 偏好：喜欢简洁代码，优先选开源方案
└─ 上次未完成：订单系统的幂等设计
━ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
```

如果检测到上次未完成的任务，还会主动提醒：

```
📎 关联提醒：上次讨论到「支付回调幂等处理」，你有提到踩过相关坑。
```

**对话过程中自动捕获关键信息**，无需手动操作：

| SOLO 自动识别 | 记录为 |
|:---|:---|
| "我喜欢""我习惯""我更倾向" | 🎯 偏好 |
| "决定了""就选""最终用" | ✅ 决策 |
| "这个项目""我的项目" | 📦 项目背景 |
| "关键是""需要注意的是""踩过坑" | 💡 经验教训 |
| "记住了""学到了" | 📖 知识点 |

**每条存储前经过"知识复利三问"过滤**（至少 2 个"是"才存）：
1. ❓ 未来会被多次用到吗？
2. ❓ 和其他知识有关联吗？
3. ❓ 用户会感谢我记住吗？

不满足的自动忽略，绝不打扰用户。

---

### 🧠 L2：OADA 深度思考 — 复杂任务自动激活

当用户提出分析/设计/决策/排查类问题时，自动启动 **OADA 闭环思考协议**：

```
O ─ Observe（观察）
   重新阐述用户需求，确认理解正确
   检索记忆中的相关经验和教训

A ─ Analyze（分析）
   拆解为子问题，至少对比 2 种方案
   引用历史决策保持一致

D ─ Decide（决策）
   给出推荐方案 + 理由
   变化时说明原因

A ─ Act（执行）
   小步实现，完成后输出下一步
```

**内置防思维陷阱机制**，智能体也能自我纠偏：

| 检测到 | 陷阱 | 自动纠正 |
|:---|:---|:---|
| 连续搜索同一文件 | 循环搜索 | 退后重新理解问题 |
| 说"试试看" | 盲目猜测 | 先调查根因再行动 |
| 说"应该能工作" | 过度自信 | 先验证再承诺 |
| 只想到一种方案 | 隧道视野 | 强制生成备选方案 |
| 忽略反面证据 | 确认偏误 | 主动寻找反例 |

---

### ✅ L3：自我验证 — 每次输出后自动自检

每次生成回答或代码后，**元认知 5 问自检** 确保交付质量：

```
□ 理解检查：我的回答真的解决了用户的问题吗？
□ 质量检查：代码/方案有没有明显的 bug 或疏漏？
□ 边界检查：是否考虑了异常情况和边界条件？
□ 记忆检查：有没有遗漏记忆中相关的信息？
□ 隐私检查：回答中是否不小心暴露了敏感信息？
```

**任何一项不通过 → 自主修正后再输出。** 你只看到正确的结果。

---

### 🌱 L4：持续进化 — 从每一次纠正中学习

当用户纠正或否定 SOLO 时：

1. ✅ 立刻承认错误，感谢纠正
2. 📝 自动将纠正内容记录为经验教训
3. 🔗 后续引用所学：

```
（根据之前你纠正过的：这个项目的数据库是 MySQL 而非 PostgreSQL，我按 MySQL 来设计）
```

**模式识别**：每累计 5 条教训记录，自动扫描是否反复在同类问题上出错。同一类错误出现 2 次，主动询问是否要调整。

### 📟 CLI 命令总览（11 个命令）

```bash
# 存储一条记忆
python3 memory_bank.py store --type <preference|decision|project|lesson|fact> --content "..." --tags "tag1,tag2"

# 智能召回（自动感知当前上下文）
python3 memory_bank.py recall --limit 5

# 搜索记忆
python3 memory_bank.py search --keyword "数据库" --limit 10

# 列出最近记忆
python3 memory_bank.py list --limit 10

# 查看统计
python3 memory_bank.py stats

# 显示可视化仪表盘
python3 memory_bank.py dashboard

# 生成 Markdown 周报
python3 memory_bank.py weekly

# 发现记忆关联
python3 memory_bank.py relation --limit 10

# 导出为 Markdown
python3 memory_bank.py export

# 清理 90 天未使用的记忆
python3 memory_bank.py cleanup

# 删除指定记忆
python3 memory_bank.py delete <mem_id>
```

## 💪 技术亮点

1. **零外部依赖** — 纯 Python 3 标准库实现，无需 pip install
2. **隐私优先** — 所有数据存储在用户本地 JSON 文件，绝不外传
3. **智能去重** — 词重叠相似度算法（阈值 0.85）避免重复存储
4. **5 层渐进激活** — 从开机仪式到持续进化，能力随使用深度自动升级
5. **知识复利指数 (KCI)** — 多维度量化知识库健康度（关联度 30% + 多样性 20% + 活跃度 20% + 基础分 10）
6. **关联发现引擎** — 自动挖掘词重叠、标签重叠、项目关联，让知识点连接成网络
7. **可视化仪表盘** — 终端内直接输出，无需任何前端依赖
8. **Markdown 周报** — 一键生成可分享周报，方便做知识复盘
9. **自动清理** — 90 天未访问的记忆自动归档，库大小可控
10. **从简原则** — 非侵入式设计，自动运行，不改变用户现有工作流

## 🧠 设计哲学

> **真正的智能，是记住不该忘记的东西。**

这个 Skill 的设计围绕三个核心理念：

1. **最小化摩擦** — 自动工作，无需用户主动操作
2. **最大化价值** — 严格过滤，只保留真正值得记住的信息
3. **保护隐私** — 用户数据用户自己掌控，绝不上传
