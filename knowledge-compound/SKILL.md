# 知识复利引擎

> 知识复利：像复利一样，知识也是越积累、越持续、价值越大。
> 每次记住一点，每次调用一点，长期下来就是巨大的认知优势。

## 一句话

让 SOLO 拥有持久记忆。你不需要再反复交代同一件事。每次对话中你分享的信息——偏好、决策、项目背景、重要事实——都会被自动记住，并在未来相关场景中主动回想起来。

## 适合谁用

- 经常和 SOLO 讨论多个项目的用户，不想每次重新交代背景
- 有特定工作偏好的用户，希望 SOLO 越用越了解自己的习惯
- 任何觉得"AI 每次会话都失忆很烦"的人

## 使用方法

你不需要学任何命令。正常聊天就行。

### 自动行为（无需触发）

| 场景 | SOLO 会做什么 |
|------|--------------|
| 你说"我喜欢/I prefer..." | 自动记住你的偏好，下次自动遵循 |
| 你做一个重要决策 | 自动记录决策和原因 |
| 你介绍一个项目/背景 | 自动提取关键信息存档 |
| 新会话开始 | 自动加载相关记忆，主动提醒你 |
| 话题切换到之前讨论过的领域 | 自动调出历史记忆，作为上下文参考 |
| 你问"我记得之前说过..." | 自动搜索并呈现相关记忆 |

### 手动管理（需要时说就行）

| 你说 | 会发生什么 |
|-----|-----------|
| "记住这个：..." | 强制保存当前信息为记忆 |
| "我有哪些记忆？" | 展示最近的记忆列表 |
| "搜索记忆：xxx" | 搜索和 xxx 相关的所有记忆 |
| "删除记忆：xxx" | 删除特定记忆（会先确认） |
| "导出记忆" | 生成一份 Markdown 格式的记忆报告 |
| "清空记忆" | 清除所有记忆（会先确认） |

## 工作方式

### 记忆捕获机制

当用户分享以下类型信息时，自动触发记忆捕获：

```
信息类型          触发词示例
─────────────────────────────────────────
个人偏好    → "我喜欢" "我习惯" "我倾向"
决策记录    → "决定了" "就用" "选这个"
项目背景    → "这个项目" "我们的产品"
重要事实    → "关键是" "需要注意"
教训经验    → "踩过坑" "之前失败" "学到了"
```

**质量过滤**：不是所有信息都值得记住。每次捕获前自问三个问题（知识复利三问）：
1. 这个信息未来会被多次用到吗？
2. 它和其他知识有关联吗？
3. 用户会感谢我记住了这个吗？

三个问题至少两个回答"是"，才记录。

### 记忆召回机制

```
触发召回的场景：
├─ 新会话启动 → 自动加载 TOP 5 最近记忆概要，主动提示
├─ 话题匹配 → 当前话题与历史记忆的标签/内容匹配时，引1-3条相关记忆
├─ 用户主动查询 → 搜索所有匹配记忆
└─ 用户做决策时 → 提醒相关历史偏好和教训
```

召回时格式：
> 📖 *我记得你之前说过……[相关内容]*
> *（记录于 YYYY-MM-DD，当时在讨论 [上下文]）*

### 记忆存储

- 所有记忆存储在 `.knowledge-compound/memory-bank.json`
- 纯 JSON 格式，用户可随时打开查看和编辑
- 自动去重：相同内容不重复存储，只更新访问次数
- 智能清理：超过 90 天未访问的记忆自动归档

## 核心行为规则

1. **非侵入**：记忆提示只在相关时出现，不会每条消息都打断用户
2. **准确引用**：回忆时必须引用原文，不能自己编造记忆内容
3. **用户可控**：用户任何时候都可以查看、修改、删除自己的记忆
4. **隐私优先**：记忆存在本地，不上传任何外部服务器
5. **从简原则**：用户不需要学任何命令，自然语言就能管理记忆

## 编写记忆条目规范

每条记忆包含：
```json
{
  "id": "mem_xxx",
  "type": "preference | decision | fact | project | lesson",
  "content": "记忆的文本内容（用户的原话或精确概括）",
  "context": "产生这条记忆的场景上下文",
  "tags": ["标签1", "标签2"],
  "created": "YYYY-MM-DD",
  "last_accessed": "YYYY-MM-DD",
  "access_count": 0
}
```

## 和我的自我进化系统的关系

本 Skill 独立于我的自我进化系统运行。它只做一件事：**记住该记住的，想起该想起的**。自我进化系统（OADA 闭环、主动建议等）是独立的能力层，两者配合但不耦合。
